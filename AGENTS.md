# cybersec-main

## Stack

- **Runtime**: Bun (package manager + dev server runner)
- **Client**: Vite + React 19 + TanStack Router
- **Server**: Cloudflare Workers (entry: `src/server.ts`)
- **UI**: shadcn/ui + Tailwind CSS (`components.json` at root)
- **Format**: Prettier — `bun run format`
- **Typecheck**: `tsc --noEmit` via `bun run typecheck`
- **Lint**: ESLint — `bun run lint`

## Commands

| Command           | What                                        |
| ----------------- | ------------------------------------------- |
| `bun run dev`     | Start dev server with HMR                   |
| `bun run build`   | Full build (client + server)                |
| `bun run check`   | `typecheck && lint` — run before committing |
| `bun run format`  | Prettier write                              |
| `bun run preview` | Build + preview via wrangler                |
| `bun run deploy`  | Build + wrangler deploy                     |

## Conventions

- **Path alias**: `@/*` → `src/*` (used in imports)
- **Route tree**: `src/routeTree.gen.ts` — auto-generated by TanStack Router; regenerate after adding/renaming route files
- **Generated files** (`*.gen.ts`) — excluded from prettier and committed as-is
- **Server file**: `src/server.ts` is the Cloudflare Worker entry. `src/start.ts` is for local dev HMR setup, imported by the server.
- **shadcn/ui**: add components with `bunx shadcn@latest add <name>`
- **Style**: `verbatimModuleSyntax` — use `import type` for type-only imports
